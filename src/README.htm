This repo is an implementation of U-Net for river extraction and depends on the following libraries:

         Keras >= 1.0

         Numpy

         Tifffile

This code should be compatible with Python versions 2.7-3.5.

Training for this model was done on a NVIDIA Tesla P100 16GB GPU.

 

Training and test phases of U-Net to extract rivers

I. Pre-processing & tiling (in Matlab):

1.      Copy the orthorectified 16-bit 8-band WordView (WV) surface reflectance imagery of the training area to Rivers/src/tiling

2.      Copy the corresponding 8-bit river mask (hand-digitized) to Rivers/src/tiling

3.      Run tiling_pretraining.m in the Rivers/src/tiling

* MAKE SURE saveastiff.m function is in the same directory as tiling_pretraining.m tiling_pretraining.m tiles the WV image and its corresponding river mask into smaller images of 800800 pixels with steps of 100 pixels, and saves the generated multi-page tils in the 'tiled multi-page image' and 'tiled multi-page river mask' folders. The entire data set is comprised of 209 image tiles and their corresponding river mask tiles.

4.      Run randselect_pretraining.m in the Rivers/src/training

randselect_pretraining.m makes a 'data' directory and randomly:

         renames and saves 80% (167 tiles) and 20% (42 tiles) of the generated multi-page image tiles in 'data/image_tiles_fortraining' and 'data/image_tiles_fortest', respectively.

         renames and saves 80% (167 tiles) and 20% (42 tiles) of the generated multi-page river mask tiles in 'data/mask_tiles_fortraining' and 'data/mask_tiles_fortest', respectively.

 

II. Training (in Python):

5.      Run train_unet.py in the Rivers/src/training

* MAKE SURE data, gen_patches.py, and unet_model.py are in the same directory as train_unet.py

Training is performed for 150 epochs (35 sec per epoch), with a batch size of 32 patches. Using a NVIDIA Tesla P100 GPU the training lasts for about 1.5 hours. Training weights are saved as 'unet_weights.hdf5' in the weights folder.

 

III. Testing (in Python):

6.      Run predict_fortest.py in the Rivers/src/training

predict_fortest.py reads 42 image tiles of 800*800 pixels from data/image_tiles_fortest and saves map of each tile showing river pixels with more than 50% probability in the 'data/mask_tiles_fortest_predicted'.

 

Prediction phase of U-Net to extract rivers

I. Prediction (in Python):

1.      Run predict.py in the Rivers/src/training

* MAKE SURE the orthorectified 16-bit 8-band WV image and its multi-page tiff are in the same directory as predict.py

** MAKE SURE to enter the name of the WorldView image in line 11 of the code:

image= normalize(tiff.imread(name of the WV image.tif').transpose([1,2,0]))

predict.py uses the trained U-Net weights to process the multi-page tiff with 800800 sliding-window with 50% overlap, and generates 800800 maps where each pixel value is the probability of the pixel belonging to the river class. The maps are saved in the data/WV_predicted.

 

II. Image mosaic (in Matlab):

2.      Run mosaic.m in the Rivers/src/training

* MAKE SURE natsort.m and natsortfiles.m functions are in the same directory as mosaic.m

mosaic.m merges the 800*800 maps of prediction in data/WV_predicted into an image-level prediction, and exports the Image-level prediction back to a GeoTiff file.
